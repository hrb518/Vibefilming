# Vibefilming 短视频制作避坑SOP
1. 叙事主体必须全程统一，禁止中途切换核心主角导致情绪承接断裂
2. 含人物画面必须规避完整正脸，多格画面生成后检查重复AI元素/乱码文字
3. 多段视频拼接遇crossfade编码错误时，优先用video_concat硬拼接替代
4. 所有生成的视觉素材（图/视频）必须过vlm_understand审核后再进入下一步
5. Seedance视频生成默认等待300s易超时，需设置max_wait=400s延长等待阈值
6. 单个镜头/entity视图返工最多2轮，2轮仍不达标直接标记best_effort使用，避免过度消耗预算
7. 链式生成多段视频时，必须每一段过审后才能生成下一段，上一段的瑕疵会被后续链式放大
8. 多段链式视频的生成prompt必须明确写明「无跳切/动作连贯/承接上一段位置」，避免出现逻辑断裂的跳帧问题
9. 涉及人物服装细节（如纯色无条纹等）时，必须将该要求单独在prompt开头重复强化，避免模型自动添加多余装饰
10. 单人场景视频生成必须在prompt中明确添加「全程仅出现1个目标人物，无其他多余人物」约束，防止生成多余的双胞胎人物
11. 运动类动作视频的prompt需要明确运动规则约束，如篮球运球场景需写明「边拍球边跑动，不得出现抱球跑动的走步违例动作」，避免不符合运动常识的逻辑错误
12. 涉及公共常识类内容（如垃圾分类、交通规则等）的公益视频，必须严格符合真实规范，避免传递错误引导信息
13. 生成整洁/干净的场景时，prompt必须明确添加「完全无任何垃圾、碎屑、杂物」的强制约束，防止模型隐性生成残留垃圾
14. 需要生成写实风格真人视频时，优先做手部/背影/远景特写镜头，规避正脸/全身人物生成，避免AI生成的人物畸形/卡通残留问题
15. ffmpeg的burn_subtitle工具对中文字符编码支持差，会出现乱码问号，需提前配置中文字体或后期手动添加字幕
16. 写实风格视频优先做无人物的场景镜头，若必须出现人物，避免展示完整面部与全身肢体，减少AI生成缺陷
17. 生成需要文字浮现的镜头时，必须明确要求文字无错别字、无多余标点，严格匹配指定内容，避免生成错误文字
18. 生成仅拉远的镜头时，必须明确写“全程只有拉远动作，无平移/移动镜头”，防止模型自动添加额外镜头运动
19. 涉及垃圾桶/公共设施等道具时，若不需要多余标识必须明确写“无任何多余文字/标识”，避免模型自动生成桶身分类文字
20. 链式生成的后续镜头若承接已完成的动作（如垃圾已经投完），必须明确禁止生成前置动作（如“无投放垃圾动作、无残留垃圾袋”），防止模型回溯生成之前的动作
21. 生成多人物场景时，若需要限定角色数量必须明确写「全程仅出现X个指定人物，无任何其他路人/陌生人/多余人物」，防止模型自动生成背景路人
22. 同一镜头多次生成仍出现不符合要求的前置片段时，优先对已有基本符合要求的视频使用video_trim裁剪冗余部分，比反复重新生成效率更高
23. 裁剪/后期修改过的视频无法作为链式参考（缺少.url.txt sidecar文件），后续镜头生成时可不传reference_video_url直接生成，避免参数错误
24. 修复成片特定时间段的问题时，优先定位到对应的单镜头修改重制，无需全片重制，大幅降低时间/预算消耗
