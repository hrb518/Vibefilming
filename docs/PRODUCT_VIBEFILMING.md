# VibeFilming · 产品文档

> 一句 Brief，一部短片。AI 导演自己分镜、自己出片、自己审、自己改。


---

## 一、产品简介

VibeFilming 是基于 **VibeFilming 运行内核 + 影视技能库**双层架构打造的全自主 AI 短片导演 agent。用户只需输入一句自然语言 brief（"做个 30 秒环保公益片"、"宠物跳舞 15 秒竖屏"、"60 秒武侠刀光剑影"），系统会自动完成：

**分镜规划 → 角色/道具/场景基准图 → 多段视频生成 → VLM 自审 → 拼片 → BGM/配音后期 → 成片交付**

整条流水线 **零写死、零编排**——agent 自己读技能库、自己拿主意、自己审片不过审就重做，把每次踩的坑沉淀成下一次的能力。

---

## 二、产品特性 / 营销点

### 1. 智能自审与自调（VLM 当导演审片，不是 QA 验收）

业界做 AI 短片普遍把 VLM 当 QA：列一张 checklist 问"是否符合 A+B+C+D"，过审就放行，不过审就报错让用户自己想。

**VibeFilming 把 VLM 当导演**：

- **导演审美题**而非 QA 验收题——问的是"作为这部片的导演你看上了哪些点 / 你觉得哪里不对 / 想要的样子是什么"
- 按产物类型分 **A 基准图 / B 单镜头 / C 全片成片** 三套审片模板，每套有专属关注重心（基准图重一致性、单镜头重镜头语言、全片重叙事节奏）
- 不过审 → VLM 给出"导演视角的问题诊断 + 想要的样子" → agent 走 **Prompt Engineering 7 步** 翻译成合规 prompt → 自动重做（最多 2 轮）

**对客户的意义**：成片质量自洽闭环，不需要客户当眼睛。

### 2. 极高自由度的底层架构（决策内核 + 技能库分离）

竞品大多是写死的 pipeline：分镜 → 出图 → 出片 → 拼片 走固定流程，调一处改一处都要工程介入。

VibeFilming 把"决策"和"经验"彻底分开：

| 层 | 内容 | 改谁 |
|---|---|---|
| 决策内核 | VibeFilming 主循环（调工具、循环、自审、记账），不钉死任何业务流程 | 不动 |
| 影视技能库 | `skills/skill_*/SKILL.md` × 8 篇 + sys_prompt 铁律 + film 工具集 | **改 md，不碰一行代码** |

**导演拍法换路线 = 改一个 markdown**：

- 想加"九宫格关键帧法" → 改 `skills/skill_storyboard/SKILL.md`
- 想换防变脸策略 → 改 `skills/skill_entity_consistency/SKILL.md`
- 想加全新技能（如"AI 配音口型对齐"）→ 在 `skills/` 下建一个新文件夹放 SKILL.md，符合 Anthropic Agent Skills 标准，**下次对话自动生效**

**对客户的意义**：业务定制和场景迁移成本接近为零。

### 3. 工业级一致性（双重防变脸）

短片最难的不是出一张好图，而是**让同一个角色在 N 段视频里长得一样**。

VibeFilming 的双重防变脸方案：

- **基准图先行**：每个角色/道具/场景先用 `gen_image` 出一张 turnaround 三视图（正/侧/背），过 VLM 审片确认后才进入下一步
- **链式参考强约束**：所有镜头出片时把基准图列表喂给 `gen_video_t2v` 的 `reference_images`，链式视频从第 2 段起 `reference_video_url` 必传上一段
- **sys_prompt 铁律 #10 链式必传**：违反就触发自检告警

**对客户的意义**：30 秒以上多镜头短片，主角不会拍着拍着换张脸。

### 4. 语言反馈精准局部重做

客户审片改稿场景下，VibeFilming 支持**自然语言精准局部重做**：

- "把第 3 个镜头里那把剑改成红色，重做这一段"
- "整体氛围太冷，最后镜头加一束暖光"
- "BGM 紧张感不够，换成鼓点节奏"

agent 会精确定位到对应资产（哪一段视频 / 哪张基准图 / 哪个 BGM），重新走 PE → 出片 → VLM 审片闭环，**不动其他资产**。

**对客户的意义**：改稿成本仅限于被改的部分，不会因为重做一段把全片打乱。

### 5. 自我进化（越拍越聪明）

每次踩的坑都会以以下三种形式沉淀：

- **sys_prompt 铁律**：连续踩 3 次以上的高频坑写进顶部 🚨 水印线（如"链式必传 / VLM 必审 / PE 必读"）
- **SKILL.md 案例库**：每个 skill 文件底部维护"踩坑案例 Case A/B/C"
- **`update_working_checkpoint`**：项目级工作笔记，agent 自己记当前进度防遗忘

**对客户的意义**：这套系统是有"职业生涯"的——你给它拍的片越多，它对你的业务调性越熟。

## 三、竞品比对

| 维度 | 通用 AI 视频生成（Sora / Runway / Pika） | 写死 pipeline 的短片工厂 | **VibeFilming** |
|---|---|---|---|
| 输入粒度 | 单镜头 prompt | 多镜头表单 | **一句 brief** |
| 多镜头一致性 | 完全不保证 | 工程预埋 | **基准图 + 链式参考双重保险** |
| 自审能力 | 无 | QA checklist | **VLM 导演审美题 + 自动重做** |
| 业务定制成本 | N/A | 改代码 | **改 md，0 代码** |
| BGM/配音 | 模型脑补（多段拼接断裂） | 独立模块 | **后期统一生成 + amix，避免断裂** |
| 自主性 | 单步生成 | 流水线编排 | **agent 自决策 + 自审 + 自重做** |

## 四、调试技巧

### 4.1 想改它的拍法

参考 `DIRECTOR_GUIDE.md` 第三节"想改它的拍法去改哪里"，对号入座改对应的 SKILL.md。

### 4.2 想看它每步在干嘛

每个项目目录下 `projects/<pid>/logs/` 有三套日志：

- `tool_calls.jsonl`：每次工具调用的 ts/tool/args/result
- `model_calls.jsonl`：每次大模型/Seedance/Seedream 调用的完整 raw（含实际发出的请求体 body 字段）
- `seedance_calls.jsonl`：视频任务的 task_id / status / 耗时 / 参考图数

### 4.3 视频排队卡顿

Seedance 集群繁忙时单 task 可能 queued 20+ 分钟，run 阶段还会再等 10+ 分钟。**这不是格式问题，是真排队**——同一个 task_id 用 `query_video_task` 重新 query 即可拿到成片（task 在云端不会因本地中断而丢失）。

### 4.4 软约束失效兜底

如果 agent 漏审参考图 / 漏读 PE skill，在 sys_prompt 顶部 🚨 开拍三件套水印线已硬约束：

- 进门必读 4 个 skill
- gen_image / gen_video_t2v 后下一步必须 vlm_understand
- 链式视频从第 2 段起 reference_video_url 必传

复盘自检三条机械可数指标：`file_read < 4 次 / gen_image 数 ≠ vlm_understand 数 / 链式段 ref=null` —— 任一命中即违规。

## 五、客户案例

### 案例 A · 30 秒环保垃圾分类公益片

- 输入："生成一段 30 秒的环保垃圾分类主题公益视频"
- 自动产出：3 个角色/道具/场景基准图 + 1 段 6 秒成片（Seedance 排队 41 分钟，最终 succeeded）
- 项目目录：`projects/p20260604_154233_*/`

### 案例 B · 多镜头链式短片

- 输入："60 秒武侠刀光剑影 5 个镜头"
- 自动产出：主角 turnaround + 兵器立绘 + 5 段链式承接视频，每段以上一段尾帧为 `reference_video_url`

### 案例 C · 九宫格关键帧并行法

- 输入："30s 咖啡广告，并行出片"
- 自动产出：一次性出 9 张关键帧 → 多段视频并行提交 → 全过审拼片，比串行快 ≈ 3-5 倍

## 六、新功能预告

- **API 批量模式**：单账号并发提交 N 个项目，统一回调
- **高一致性 vs 高创意性配置**：开放底层 seed/temperature/参考图权重三组档位，客户按场景选
- **多模型路由**：Seedream/Seedance 之外接入更多文生图/文生视频底模，agent 按 brief 类型自动路由
- **声音侧扩展**：TTS 唇形同步 + 多说话人分轨
